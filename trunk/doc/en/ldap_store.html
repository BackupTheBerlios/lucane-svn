Using LDAP for store backend

What is the store ?

Lucane handles users, groups, plugins and services. These elements are called 'concepts'. Access management is represented as links between groups and other concepts.
These concepts and links are used internally as objects, which have to be stored somewhere : database, ldap, ...
To sum up, the store is the persistance layer between Lucane concepts.

Why use LDAP for the store ?

If you want to use the same user accounts for multiple programs, LDAP is probably a good idea. If you only want to use Lucane, you'd better use to the database backend.

How to configure the LDAP backend ?

First, set the backend to 'ldap' in server-config.xml.
The configuration file for the ldap backend is ldap-store.xml

Set your ldap server in the 'url' attribute.
You have to change the authentication tag to set your bind-dn and password.

Each concept has the same configuration structure :

  • dn : the ldap path to access the concept root
  • attributes : the static attributes to set to newly created concepts
  • mapping : the mapping between a concept attribute in Lucane and a field in LDAP

Using this mapping, we don't need a custom schema.

Generating initial structure

A default ldif is available : init-ldap.ldif

You will have to edit this ldif to match your directory structure.
