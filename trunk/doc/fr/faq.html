Foire Aux Questions

Introduction and informations générale

  • Qu'est ce que Lucane ?

Lucane est une plate-forme collaborative. Cela signifie que ses utilisateurs ne sont pas strictement définis. Cette plate-forme devrait être utile pour tout groupe de travail, avec des petites choses à ajouter pour l'adapter à un domaine spécifique.

  • Qu'est ce qu'une plate-forme collaborative ?

Pour faire court, c'est un groupware qui peut être agrandi avec de nouvelles applications développées pour des domaines spécifiques.

  • Pourquoi utiliser Lucane ?

La motivation pouvant mener à son utilisation est la volonté de travailler en équipe avec quelque chose de plus puissant qu'une simple messagerie. Cela peut être utile à tout groupe qui désire travailler en collaboration.

  • Où est ce que je commence ?

Vous pouvez commencer par installer Lucane sur votre ordinateur et l'utiliser avec vos amis et co-travailleurs. Vous pourrez utiliser la messagerie instantanée, le partage de fichiers, les forums et plus encore. Quand vous vous sentirez prêt, vous pourrez utiliser le système de workflow pour gérer votre projet, et même écrire vos propres applications.

  • Lucane fonctionnera-t-il sur mon ordinateur ?

Probablement. La plate-forme est écrite entièrement en Java, donc si votre ordinateur utilise des applications Java, Lucane s'exécutera sans aucun problème. Pour être plus précis, tout ordinateur récent avec une JVM devrait aller.

  • Est ce un logiciel libre ?

Oui. Lucane est protégé par la licence GNU LGPL. Pour être bref, celle-ci vous permet d'utiliser Lucane, de le redistribuer, de le modifier et de partager vos modifications. La seule restriction is que si vous modifiez la plate-forme elle-même, vous devrez placer vos modifications sous la même licence.

Récupérer Lucane

  • De quoi ai-je besoin ?

Vous n'avez besoin que de Java et d'une base de données. Lucane est testé avec MySQL (http://www.mysql.com) et Postgres (http://www.postgres.org). Si vous ne voulez pas installer une base de données, Lucane est livré avec HSQLdb qui ne requiert pas d'installation.

  • Comment est ce que je le compile ?

Vous aurez besoin de Ant (http://ant.apache.org) avec bsf.jar et js.jar pour compiler Lucane. Récupérer les sources à partir du site web et lancez simplement ant

  • Je ne veux rien compiler. Y a-t-il des binaires ?

Évidemment. Vous les trouvez à côté des sources la section Téléchargement du site web.

Utiliser Lucane

  • Qu'est ce qu'un plugin ?

Un plugin est un composant client développé pour notre plate-forme. Quelques plugins sont livrés avec Lucane, comme le système de messagerie et le partage de fichier.

  • Qu'est ce qu'un service ?

Un service est la partie serveur d'une application Lucane. Quelques applications marche en peer to peer, d'autres utilisent un style client/serveur plus classique. Ce dernier utilise un service pour accéder au ressources du serveur. Un exemple est le forum, qui utilise un service pour récupérer les messages depuis la base de données du serveur.

  • Comment cela fonctionne-t-il ?

Plutôt bien. Le système de plugin/service est comme le systèmeapplet/servlet system utilisé dans un environnement web, avec moins de limitations, puisqu'un plugin ne peut être chargé contre votre volonté, nous n'avons pas besoin de resteindre ses capacités, contrairement aux applets.

  • Où puis-je trouver des applications supplémentaires ?

Nous espérons que ceux qui écrivent des applications aimeront les partager avec la communauté. Dans ce cas, nous les rendrons disponibles sur notre site web.

Développer avec Lucane

  • Puis je écrire de nouvelles applications ?

Bien sûr ! Lucane a été conçu avec cette idée en tête, alors pourquoi ne pas l'utiliser ?

  • Où puis je héberger mes applications ?

Pour le moment, il n'y a pas de page du genre de freshmeat pour vos applications. Vous pouvez nous les envoyer et nous les hébergerons. Si vous préférez les héberger vous-même, envoyez nous simplement une url et nous ferons un lien.

  • Y a-t-il une documentation sur l'écriture d'extensions ?

Oui. Et ce n'est ni long ni compliqué. Vous pouvez la lire ici, and use the api documentation to use Lucane at its best.
