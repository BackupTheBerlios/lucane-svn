Guide de démarrage

Lucane est fait de plusieurs modules :
  • un client
  • un serveur
  • un proxy

Mise en place du client

Il n'y a qu'un seul fichier de configuration

  • client-config.xml

C'est un fichier xml auto-décrit. Si vous voulez seulement tester lucane avec le client et le serveur sur le même hôte, aucune modification n'est nécessaire.

Voici une description de tous les paramètres : (un champ en gras signifie qu'il est obligatoire)

  • server : le serveur à utiliser (hôte et port)
  • login : le login par défaut à afficher dans la boîte de connexion
  • language : la locale à utiliser (en, fr, ..)
  • proxy : seulement utile si vous n'êtes pas sur le même réseau que le serveur
  • public-ip : seulement utile pour positionner l'interface réseau à utiliser si vous en avez plus qu'une

Une fois que le fichier de configuration est personnalisé, vous pouvez démarrer le client avec client.sh ou client.bat en fonction de votre envirronnement. Le serveur doit évidemment être connecté.

Le client téléchargera automatiquement les applications que vous pouvez utiliser. Elles seront mises à jour à chaque fois que ce sera nécessaire sans aucune intervention au niveau du client.

Mise en place du serveur

Le serveur est aussi simple à installer, le fichier de configuration principal est :
  • etc/server-config.xml

Voici une description de tous les paramètres : (un champ en gras signifie qu'il est obligatoire)

  • database
    • jdbc : le driver à utiliser
    • dblayer : la couche d'abstraction de la base de données à utiliser
    • pool : les options du pool de connexions
  • store : le backend à utiliser pour stocker les utilisateurs, les groupes, les plugins et les services
  • authenticator : la classe a utiliser pour gérer l'authentification

Les moteurs de base de données testés sont HSQLdb, MySQL et Postgres. Les autres n'ont pas été testés et ne sont pas activés dans la distribution par défaut.
Si vous ne voulez pas utiliser une base de données particulière, laissez le fichier de config tel quel pour utiliser HSQLdb.

Le sotr peut utiliser la base de données ou un serveur LDAP. Pour utiliser ldap, regardezle guide de sauvegarde ldap.

Il y a deux authenticators :

  • org.lucane.server.auth.DefaultAuthenticator : Rien de particulier
  • org.lucane.server.auth.LockingAuthenticator : bloque les comptes utilisateurs après 3 échecs.

Vous pouvez démarrer le serveur par server.sh ou server.bat

Mise en place du proxy

Vous aurez besoin du proxy seulement si vous avez plusieurs réseaux qui ne sont pas directement visibles. C'est le cas si vous utilisez le NAT par exemple. Le proxy ne nécessite aucune configuration.

Configuration du système

Pour administrer le système nouvellement installé, vous devrez vous connecter avec le nom d'utilisateur admin et le mot de passe admin. La première chose à faire sera de changer ce mot de passe. Vous pourrez démarrer l'interface d'administration et créer des utilisateurs, des groupes, et leur donner les droits qu'ils nécessitent. Regardez, svp, l'aide de l'interface d'administration pour plus d'information. C'est plutôt simple.

Comptes utilisateur par défaut

Plusieurs comptes sont disponibles afin de pouvoir tester :

  • admin : admin
  • granted : granted
  • guest : guest
