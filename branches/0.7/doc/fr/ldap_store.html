Utiliser LDAP comme backend pour le Store

Qu'est ce que le Store ?

Lucane gère des utilisateurs, des groupes, des plugins et des services. Ces éléments sont appelés des 'concepts'. La gestion d'accès est représentée comme des liens entre des groupes et d'autres concepts.
Ces concepts sont et ces liens sont utilisés en interne comme des objets qui doivent être stockés quelque part : base de données, ldap, ...
Pour résumer, le Store est la couche de persistance pour les concepts Lucane.

Pourquoi utiliser LDAP pour le store ?

Si vous voulez utiilser le même compte utilisateur pour plusieurs programmes, LDAP est probablement une bonne idée. si vous voulez seulement utiliser Lucane, vous feriez alors mieux d'utiliser le backend de la base de données.

Comment configurer le backend LDAP ?

Premièrement, positionnez le backend à 'ldap' dans server-config.xml.
Le fichier de configuration pour le backend LDAP est ldap-store.xml

Indiquez votre serveur ldap dans l'attribut 'url'.
Vous devez modifier les attributs bind-dn et password de la balise authentication.

Chaque concept a la même structure de configuration :

  • dn : le chemin ldap pour accéder la racine du concept
  • attributes : les attributs statiques à indiqer pour les nouveaux concepts créés
  • mapping : le mapping entre un attribut de concept dans Lucane et un champ dans LDAP

En utilisant ce mapping, nous n'avons pas besoin d'un schema particulier.

Générer la structure initiale

Un ldif par défaut est disponible : init-ldap.ldif

Vous aurez surement a éditer ce fichier selon la structure de votre annuaire.
